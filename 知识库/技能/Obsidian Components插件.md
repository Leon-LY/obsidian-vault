# Obsidian Components 插件实操指南

> 一句话：Components 是未上架插件市场但口碑极高的 T0 级插件，能搭建 Notion 风格的主页、日记、书库等数据仪表盘，操作简单但上限极高。

## 插件定位

| 维度 | 说明 |
|------|------|
| 上架状态 | 未上架 Obsidian 社区插件市场 |
| 获取方式 | 公测版（免费，已停更）/ 正式版（买断制，B 站工坊或小红书购买） |
| 当前版本 | 即将发布 2.0，价格将上调 |
| 核心能力 | 主页搭建、数据视图、统计图表、热力图、画廊视图、日历视图、任务看板 |
| 学习门槛 | 低 — "有手就行" |

## 一、主页搭建（六步上手）

### 第一步：创建组件容器
1. 安装 Components 插件 → 命令面板输入 "Components" → **创建组件**
2. 输入组件名，回车
3. 点击"添加组件"打开设置面板 → 将布局切换为**网格布局**

### 第二步：添加时间组件（封面）
- 添加时间组件 → 隐藏秒 → 启用"农历"
- 样式设置：关闭边框 → 添加背景图片 → 调整字体颜色适配背景
- 拖拽组件右下角**拉满整个宽度**，作为页面封面

### 第三步：添加统计数字组件
| 指标 | 配置 |
|------|------|
| 全库字数 | 统计数据 → "属性值" → 属性选择"文件词数"。若数字大：统计类型改"比值"、分子填 10000、后缀加"万字" |
| 待办任务数 | 新建统计数字组件 → 选择对应属性 |
| 总文件数 | 新建统计数字组件 → 选择对应属性 |

### 第四步：添加热力图（图表组件）
1. 创建组件类型 → **图表**
2. 将默认柱状图切换为**任务图**（热力图）
3. 筛选功能 → 筛选到所有日记文件
4. 一期属性选择"文件名"，统计数据选择"属性值" → 属性选择"文件词数"
5. 样式设置中调整颜色

### 第五步：创建数据视图管理待办
1. 创建数据视图 → 默认表格视图，可改为**画廊视图**
2. 筛选到所有日记笔记 → 视图设置中封面设为"无"
3. 属性中添加属性（如 `task`，避免占用已有属性名）
4. 属性类型切换为**任务清单** → 勾选"显示任务面板"
5. 数据视图自动列出日记中所有待办 → 可在属性区完成/添加/删除任务

### 第六步：标签页布局组合多视图
1. 创建组合组件（默认标签页布局）
2. 将数据视图通过"跨组合移动"移入
3. 右键标签页 → 编辑 → 复制数据视图组件
4. 将副本视图类型切换为**日历** → 日期属性选择"文件名"

## 二、典型应用场景

### 日记主页
```
┌─────────────────────────────────────┐
│  日历视图（点击日期跳转日记）        │
├─────────────────────────────────────┤
│  待办事项列表（来自所有日记的 task） │
├──────────────────┬──────────────────┤
│  日记画廊视图     │  "去年今日"画廊   │
│  （浏览日记内容）  │  （高级查询语法） │
└──────────────────┴──────────────────┘
```

- **去年今日**：使用高级查询语法回顾过往日记，体现 Components 在数据筛选上的上限

### 阅读书库
- 画廊视图展示书籍笔记，封面自动提取笔记中第一张图片
- 封面位置可调整为左侧，视觉更好
- 配合时间轴组件按时间顺序展示阅读记录

### 实用型主页
- 统计数字 + 热力图 + 任务列表，页面紧凑功能性强
- 偏实用主义而非视觉炫酷

## 四、与现有首页方案的对比

本 vault 当前首页使用 Dataview + CSS 方案，参见 [[Obsidian首页设计]]。Components 的优势在于：

| 能力 | Dataview + CSS | Components |
|------|:--:|:--:|
| 固定侧栏导航 | ❌ | ✅ |
| 任务看板（拖拽） | ❌ | ✅ |
| 热力图 | ❌ | ✅ |
| 画廊视图 | ❌ | ✅ |
| 日历视图 | 需 Calendar 插件 | 内置 |
| 时间组件 | ❌ | ✅ |
| 配置复杂度 | 需写代码 | 纯 GUI |

> 迁移建议：若本 vault 未来需要更强的交互能力（任务看板、热力图、固定侧栏），可考虑迁移到 Components 方案。
