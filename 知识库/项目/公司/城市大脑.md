# 城市大脑

> 一句话：城市级数据驱动的综合治理数字化平台，实现"一屏观全域、一网管全城"。

## 概述

城市大脑是智慧城市建设的核心数据平台，以 GIS 地理信息为底座，整合政务、经济、民生、物联等多源数据，通过数据可视化技术展示城市运行状态，为政府决策和基层治理提供支撑。

荣成市城市大脑由 [[荣成大数据中心]] 主导建设，在原"数字荣成综合管理平台"基础上进行了创新性重构，当前正处于从 **1.0（数据汇聚+基础展示）向 2.0（好用+用数据）跨越** 的关键节点。

## 荣成市城市大脑现状

### 平台架构

```
支撑子系统层：GIS地理信息系统 + 数据可视化效果库 + 数据融合治理系统
          ↓
应用功能层：六大主题板块
          ↓
对外接口层：大数据中心平台、视频管理平台、经济运行直报系统
```

### 现有六大模块

| 模块 | 定位 | 数据状态 |
|------|------|----------|
| 数字底座 | 数据资产仪表盘 | 正常18项，停更6项 |
| 数字惠民 | 人口/医疗/养老/信用 | 正常15项，停更7项 |
| 经济运行 | GDP/税收/工业/金融/旅游（最丰富） | 正常55项，**停更22项** |
| 城市管理 | 垃圾分类/环卫/交通 | 正常8项，停更8项 |
| 民生诉求 | 工单全流程/考核（质量最好） | 全部正常~30项 |
| 物联感知 | 物联网设备运行 | **95%停更**（几乎空壳） |

### 存在的问题

1. **数据"僵尸化"**: 392 项数据指标中约 52 项停更（停更率 13.3%），物联感知模块 95% 停更
2. **基层治理缺失**: 功能以宏观展示为主，缺少面向社区/网格的业务工具
3. **移动端薄弱**: 独立 App 功能简单，无法支撑移动办公
4. **跨部门壁垒**: 人口库/法人库因授权问题中断，多类数据因部门调整不再提供
5. **应急联动不足**: 应急指挥系统与城市大脑数据无法互通

## 2.0 升级项目（2026年5月）

[[荣成大数据中心]] 于 2026 年 5 月启动城市大脑升级改造项目，建设周期 6 个月，新增以下功能：

### 社会治理子系统

以"[[#金梭计划]]"和"五位一体"社区治理共同体为牵引，打造基层治理数字化转型工具：

- **商户空间可视化治理**: 天地图商户分布总图，红/黄/蓝三级差异化治理
- **四级空间关联**: 镇街 → 社区 → 网格 → 商户
- **四个部门数据打通**: 自然资源局 / 住建局 / 市场监管局 / 综合执法局
- **走访管理闭环**: 发现→上报→转派→处置→反馈

### 应急指挥子系统

构建 500 米至 5 公里范围资源自动关联的智能辅助决策模块。

### 零碳大脑子系统（初步规划）

碳排放监测与分析基础框架。

### 数字人解说子系统

AI 虚拟数字人，实现数据自动讲解功能。

### 升级技术栈

前端: D3.js / ECharts / Highcharts / AntV · 后端: Java J2EE
地理空间底座: 天地图·山东
