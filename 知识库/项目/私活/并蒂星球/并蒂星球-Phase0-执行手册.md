## 🔴 BLOCKER：微信小程序类目问题——结论

### 关键发现

**好消息：个人开发者可以直接上架！**

经过对微信官方文档的核查，结论如下：

| 类目 | 个人能否申请 | 原因 |
|:---:|:---:|------|
| **工具 > 健康管理** | ✅ **可以！** | 官方明确列出"身高、体重、身体管理记录等"属于个人开放类目 |
| 工具 > 信息查询 | ✅ 可以 | 信息查询、订单功能等 |
| 工具 > 备忘录 | ✅ 可以 | 待办事项记录功能 |
| 教育 > 儿童教育 | ❌ 不行 | 需企业主体+办学许可证 |
| 生活服务 | ❌ 大部分不行 | 仅极少数子类目对个人开放 |

**并蒂星球的类目选择：`工具 > 健康管理`**

理由：
- 核心功能是"双胞胎的身高/体重/喂养/睡眠记录"，完全符合"身体管理记录"
- 个人开发者无需任何特殊资质
- 不涉及"在线问诊""医疗器械""健康咨询"等需要资质的敏感功能

### 注意事项

1. **小程序名称中避免使用"育儿""医生""医疗""儿童"等敏感词**——用"并蒂星球"或"双宝记录"这类中性名称
2. **小程序简介描述**：侧重"成长记录工具"而非"育儿指导平台"
3. **AI 育儿建议必须加免责声明**："本工具提供的信息仅供参考，不构成医疗建议"
4. 备案时**服务内容标识**选"工具-健康管理"

> ⚠️ 后续如果要上架 iOS App，Apple 对"医疗/健康"类 App 审核更严。但那是 Phase 3 的事。

## 🟢 WHO 生长标准数据——准备就绪

已获取完整的 WHO Child Growth Standards LMS 参数（0-24 个月），可直接用于代码。

### 计算公式

```
Z-score = ((X/M)^L - 1) / (L × S)   当 L ≠ 0
Z-score = ln(X/M) / S               当 L = 0

百分位 = Φ(Z-score) × 100  （标准正态累积分布）
```

> Φ() 可用 JavaScript 的误差函数近似实现，或引入简单的统计库。

### 数据覆盖

| 指标 | 性别 | 月龄范围 | 数据点 | 状态 |
|------|:---:|:---:|:---:|:---:|
| 体重 (Weight-for-Age) | 男/女 | 0~24 月 | 25×2 | ✅ 已获取 |
| 身长 (Length-for-Age) | 男/女 | 0~24 月 | 25×2 | ✅ 已获取 |
| 头围 (Head Circumference) | 男/女 | 0~24 月 | 待补充 | ⬜ |
| 身高 (Height-for-Age) | 男/女 | 24~60 月 | 待补充 | ⬜ |
| 体重/身长比 (Weight-for-Length) | 男/女 | 0~24 月 | 待补充 | ⬜ |

> 前两项已足够做 MVP 的生长曲线 Demo。数据源：WHO 官网 + CDC 数据文件。

### 双胞胎对比曲线实现要点

```
输入：两个宝宝的 (月龄, 体重/身长, 性别)
输出：ECharts 图表
  - X轴：月龄（0-24）
  - Y轴：体重(kg) 或 身长(cm)
  - 曲线1：宝宝A实测数据（散点+连线）
  - 曲线2：宝宝B实测数据（散点+连线）
  - 阴影区间：P3~P97（WHO标准），可额外标注 P50 中线
  - 差异标注：两个宝宝数据点的差值超过阈值时高亮提示
```

关键代码路径（建议单独抽一个 `utils/whoGrowth.ts` 模块）：
1. `getLMS(gender, ageMonths)` → 返回 {L, M, S}
2. `calcZScore(value, L, M, S)` → 返回 z-score
3. `zScoreToPercentile(zScore)` → 返回百分位
4. `getPercentileCurve(gender, indicator, percentile)` → 返回曲线的 (x, y) 坐标数组

## 🛠️ 技术环境搭建指南

### Step 1：创建腾讯云 CloudBase 环境

```bash
# 1. 访问 https://cloud.tencent.com/product/tcb
# 2. 开通环境 → 选择"按量计费"（开发阶段几乎免费）
# 3. 环境名称：twin-planet-dev
# 4. 地域：上海
# 5. 记录环境 ID（格式：xxx-xxxxx）

# 安装 CloudBase CLI
npm i -g @cloudbase/cli

# 登录
tcb login

# 创建环境（如已通过控制台创建则跳过）
tcb env:create twin-planet-dev
```

### Step 2：初始化 uni-app 项目

```bash
# 方案A：CloudBase 官方模板（推荐）
tcb pull -s uniapp --output twin-planet

# 方案B：HBuilderX 手动创建
# 文件 → 新建 → 项目 → uni-app → Vue3 版本

cd twin-planet
npm install

# 安装 CloudBase 适配器
npm install @cloudbase/js-sdk @cloudbase/adapter-uni-app

# 安装 ECharts（分包使用）
npm install echarts
```

### Step 3：配置 CloudBase SDK

```typescript
// src/utils/cloudbase.ts
import cloudbase from '@cloudbase/js-sdk';
import adapter from '@cloudbase/adapter-uni-app';

cloudbase.useAdapters(adapter, { uni });

export const app = cloudbase.init({
  env: '你的环境ID',
});

export const auth = app.auth({ persistence: 'local' });
export const db = app.database();
```

### Step 4：小程序后台配置

微信公众平台 → 开发管理 → 服务器域名 → 添加：

| 类型 | 域名 |
|------|------|
| request | `https://tcb-api.tencentcloudapi.com` |
| request | `https://你的环境ID.service.tcloudbase.com` |
| uploadFile | `https://cos.ap-shanghai.myqcloud.com` |
| downloadFile | `https://你的环境ID.tcb.qcloud.la` |

### Step 5：运行开发环境

```bash
npm run dev:mp-weixin
# 用微信开发者工具打开 dist/dev/mp-weixin
```
