# 在线预约咨询网站

> **一句话**：用 React + Next.js 14 做一个通用型在线预约咨询平台，补齐技术短板的同时产出可直接展示的真实案例。
>
> **首发场景**：装修公司设计咨询预约。
>
> **总工期**：2 周（MVP）。

---

## 一、项目定位

### 为什么做这个

| 目的 | 说明 |
|------|------|
| 补齐 React 生态 | 你的技术栈缺 Next.js/React 这一支，私活第一大需求 |
| 有真实案例 | 做出来就是活的项目，直接放作品集最前面 |
| 客户看得懂 | 预约咨询是通用场景，比"数据大屏"更容易让普通客户理解 |
| 可复用模板 | 做完一次，装修公司/摄影/健身/咨询各行业都能改改就用 |
| 展示全栈能力 | 访客端 + 管理后台 + 数据库 + 认证 + 部署，完整闭环 |

### 首发场景：装修公司

选择**装修公司设计咨询预约**作为第一个行业模板，原因：

- 行业利润高，客户愿意为获客工具付费
- 决策链短（装修公司老板一个人说了算）
- 与你已有的"方外设计"项目经验呼应
- 页面调性偏设计感，视觉上容易出彩

### 后续可以扩展的行业

摄影工作室 / 健身教练 / 法律咨询 / 财务顾问 / 美容美发 / 心理咨询

## 三、技术架构

### 技术栈

| 层 | 技术 | 版本 | 选型理由 |
|----|------|------|----------|
| **框架** | Next.js | 14+ (App Router) | React 官方推荐，SSR + API Routes 一体化 |
| **语言** | TypeScript | 5.x | 必选，所有现代项目的标准配置 |
| **样式** | Tailwind CSS | 3.x | 工具类优先，开发效率极高 |
| **UI 组件库** | shadcn/ui | latest | 基于 Radix UI，无障碍友好，可自定义主题 |
| **表单** | React Hook Form + Zod | latest | 高性能表单 + 类型安全校验 |
| **数据库** | PostgreSQL | 15+ | 你的强项，稳定可靠 |
| **ORM** | Prisma | 5.x | 类型安全，和 TS 天生一对 |
| **认证** | NextAuth.js | 4.x | 支持多种登录方式，Next.js 生态首选 |
| **日期处理** | date-fns | 3.x | 轻量日期库，无 Moment.js 的臃肿 |
| **部署** | Docker + Nginx 反代 + 自有服务器 | — | 你有自己的服务器，方案自选。推荐 Docker Compose 一键部署 |
| **数据库** | PostgreSQL（自有）或 MySQL | — | 你有自己的数据库，按你服务器上现用的来。推荐 PostgreSQL |

## 四、数据库设计

### 完整 Schema（Prisma）

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}

// ─── 服务/套餐 ───
model Service {
  id          String    @id @default(cuid())
  name        String                        // 服务名称，如"全案设计咨询"
  description String                        // 详细描述，支持 HTML
  summary     String                        // 简短摘要，首页卡片展示
  price       Decimal   @db.Decimal(10, 2)  // 价格，如 199.00
  duration    Int                            // 时长（分钟），如 60
  imageUrl    String?                        // 服务展示图片
  category    String                        // 分类：设计咨询/施工咨询/验收服务
  isActive    Boolean   @default(true)      // 上下架状态
  sortOrder   Int       @default(0)         // 排序
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt

  bookings    Booking[]
}

// ─── 预约记录 ───
model Booking {
  id         String    @id @default(cuid())
  serviceId  String
  clientId   String
  date       DateTime                      // 预约日期（不含时间）
  startTime  String                        // 开始时段，如 "09:00"
  endTime    String                        // 结束时段，如 "10:00"
  status     BookingStatus @default(PENDING)  // 状态
  note       String?                       // 客户备注
  adminNote  String?                       // 管理员备注
  createdAt  DateTime  @default(now())
  updatedAt  DateTime  @updatedAt

  service    Service   @relation(fields: [serviceId], references: [id])
  client     Client    @relation(fields: [clientId], references: [id])

  @@index([date])
  @@index([status])
  @@index([serviceId])
}

enum BookingStatus {
  PENDING     // 待确认
  CONFIRMED   // 已确认
  COMPLETED   // 已完成（到店）
  CANCELLED   // 已取消
  NO_SHOW     // 未到店
}

// ─── 客户 ───
model Client {
  id        String    @id @default(cuid())
  name      String                        // 姓名
  phone     String                        // 手机号
  email     String?                       // 邮箱
  address   String?                       // 小区地址（装修场景）
  source    String?                       // 来源：网站/朋友推荐/抖音
  note      String?                       // 客户备注
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt

  bookings  Booking[]

  @@unique([phone])                       // 一个手机号一个客户
  @@index([phone])
  @@index([name])
}

// ─── 时段设置 ───
model TimeSlot {
  id        String    @id @default(cuid())
  dayOfWeek Int                            // 0=周日, 1=周一 ... 6=周六
  startTime String                        // 如 "09:00"
  endTime   String                        // 如 "12:00"
  isActive  Boolean   @default(true)
  createdAt DateTime  @default(now())
  updatedAt DateTime  @updatedAt

  @@unique([dayOfWeek, startTime])         // 一天同一时段不重复
}

// ─── 休息日 ───
model Holiday {
  id        String    @id @default(cuid())
  date      DateTime  @unique              // 哪一天休息
  reason    String?                        // 原因：春节/年假等
  createdAt DateTime  @default(now())
}

// ─── 站点设置（key-value）───
model SiteSetting {
  id    String @id @default(cuid())
  key   String @unique                     // site_name / site_logo / contact_phone ...
  value String
}
```

### 数据关系图

```
Service (1) ──── (N) Booking (N) ──── (1) Client

TimeSlot  ← 独立，控制哪些时段可选
Holiday   ← 独立，控制哪些日期不可选
SiteSetting ← 独立，控制网站文案
```

### 可用时段计算逻辑（重点）

`GET /api/bookings/available-slots` 的核心算法：

```
输入：date, serviceId
输出：可用时段列表 ["09:00", "09:30", "10:00", ...]

逻辑：
1. 查 date 是否是休息日（Holiday 表）→ 是则返回空数组
2. 查 date 对应 dayOfWeek 的可用时段配置（TimeSlot 表）
3. 根据 service 的 duration（如 60 分钟），按 interval（30 分钟）生成候选时段
4. 查该 date 已有的预约记录，排除已被占用的时段
5. 返回剩余的可用时段列表
```

## 七、组件交互与状态管理

### 7.1 状态管理策略

**原则**：尽量用 Next.js Server Components + URL 参数做状态。客户端状态仅限交互类场景。

| 场景 | 方案 |
|------|------|
| 服务列表筛选 | URL search params (`/services?category=design`) |
| 预约多步表单 | React Hook Form + FormContext（客户端状态） |
| 日历选择 | 本地 useState + URL 同步 |
| 管理端路由 | 服务端 Layout 组件 + 客户端 active 高亮 |
| 全局数据（设置） | React Context（仅在管理端使用） |

### 7.2 预约多步表单状态管理

```typescript
// 路径: /booking/[serviceId]
// 状态: URL query ?step=1&date=2026-06-15&time=09:00

// 1. URL 驱动当前步骤
const searchParams = useSearchParams();
const currentStep = Number(searchParams.get('step')) || 1;

// 2. React Hook Form 管理表单数据
const form = useForm<BookingFormData>({
  resolver: zodResolver(bookingFormSchema),
  defaultValues: {
    name: '',
    phone: '',
    address: '',
    note: '',
  },
});

// 3. 步骤间导航用 router.push 更新 URL
const nextStep = () => {
  const params = new URLSearchParams(searchParams);
  params.set('step', String(currentStep + 1));
  router.push(`/booking/${serviceId}?${params.toString()}`);
};
```

### 服务器现状

|                  |      |                                            |

### 新项目部署策略

**使用子路径 `/booking`，不占新域名、不占新端口、不动已有配置：**

```
49.232.49.175/         → 已有配置，完全不动

49.232.49.175/booking  → 新增 Nginx location
                              ↓
                        proxy_pass 到 localhost:3002
                              ↓
                        Docker 容器（Next.js 运行在 3002）
```

### 部署步骤

#### 1. Next.js 配置（子路径支持）

```javascript
// next.config.js
const nextConfig = {
  basePath: '/booking',
  assetPrefix: '/booking',
};

module.exports = nextConfig;
```

#### 2. Docker Compose

```yaml
# docker-compose.yml
version: '3.8'
services:
  booking-app:
    build: .
    ports:
      - "127.0.0.1:3002:3000"
    environment:
      - DATABASE_URL=postgresql://postgres:password@investlearn-db:5432/booking_app
      - NEXTAUTH_SECRET=your-secret-here
      - NEXTAUTH_URL=http://localhost:3002/booking
    restart: always
```

#### 3. Nginx 新增配置（不动已有）

新建 `/etc/nginx/sites-enabled/booking.conf`，独立文件，不影响已有站点：

```nginx
location /booking {
    proxy_pass http://127.0.0.1:3002;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    rewrite ^/booking(/.*)$ $1 break;
}

location /booking/_next/static {
    proxy_pass http://127.0.0.1:3002/_next/static;
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

#### 4. 数据库

直接用已有的 PostgreSQL 容器：

```bash
docker exec -it investlearn-db psql -U postgres -c "CREATE DATABASE booking_app;"
```

连接字符串：
```
DATABASE_URL=postgresql://postgres:postgres@investlearn-db:5432/booking_app
```

#### 5. 部署命令

```bash
cd /path/to/booking-app
git pull
docker-compose up --build -d
nginx -s reload
```

访问 `http://49.232.49.175/booking` 即可看到。

### Docker Compose 配置

```yaml
# docker-compose.yml（项目根目录）
version: '3.8'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgresql://user:password@host:5432/booking_app
      - NEXTAUTH_SECRET=your-secret-key
      - NEXTAUTH_URL=https://booking.your-domain.com
    restart: always
```

### Nginx 配置

```nginx
# /etc/nginx/sites-available/booking.conf
server {
    listen 80;
    server_name booking.your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl;
    server_name booking.your-domain.com;

    ssl_certificate /etc/nginx/ssl/your-cert.pem;
    ssl_certificate_key /etc/nginx/ssl/your-key.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 图片上传目录
    location /uploads {
        alias /var/www/booking-app/uploads;
        expires 30d;
    }
}
```
