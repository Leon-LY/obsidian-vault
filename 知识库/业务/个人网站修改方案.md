# 个人网站修改方案

> 一句话：把"找工作"的工程师网站，改成"可合作"的独立开发者网站。高级感、不廉价。

## 使用方法

直接把本页内容发给 AI（Claude / Cursor / 通义千问），让它按此执行修改。最后加一句：

> 注意：所有文案要保持高级感、专业感。不使用"私活"、"接单"、"¥XXX 起"、"免费"等措辞。语言风格偏正式但不生硬，像一家小型技术咨询公司的官网调性。

### 4.2 作品描述模板

每件作品统一按这个结构重写：

```
[项目类型] · [技术栈]

▎项目概述
一句话说明项目背景和定位。

▎核心能力体现
- 2-3 条，说明解决了什么问题
- 用数据说话（服务了多少用户、覆盖范围、效果指标）

▎技术方案
简写，1-2 行带过即可。
```

### 4.3 示例文本

**城市大脑：**

```
数据可视化 · ECharts / DataV / WebSocket

▎项目概述
荣成市城市大脑是市政府指挥中心的核心展示系统，为市领导决策和应急指挥提供实时数据支撑。

- 20+ 数据维度实时监控，涵盖经济、民生、交通、环境等核心领域

▎技术方案
基于 ECharts 和 DataV 构建可视化层，后端 Java 支撑数据采集与推送。
```

**智慧社区：**

```
移动端应用 · Vue 3 / Vite / Pinia / Vant UI

▎项目概述
面向荣成市全体社区居民的一站式数字化服务平台，内嵌于爱山东 App。

▎核心能力体现

▎技术方案
基于 Vue 3 + Vite 构建 H5 应用，配合 Pinia 状态管理和 Vant UI 组件库。
```

**方外设计：**

```

▎项目概述

Nuxt 4 + Vue 3 前端，Express + PostgreSQL 后端 API，Docker + Nginx 部署。
```

### FAQ 文案（如有）

**Q：合作流程是怎样的？**
A：需求沟通 → 方案评估 → 报价与排期 → 开发交付 → 验收上线。

**Q：支持哪些类型的项目？**
A：Web 全栈应用、数据可视化大屏、H5 / 小程序、系统架构设计与改造。可根据具体需求评估。

**Q：如何保障项目质量和进度？**
A：按里程碑分阶段交付，定期同步进度，验收确认后进入下一阶段。
