### 🥉 shadcn/improve

| 维度 | 详情 |
|------|------|
| **作者** | shadcn（shadcn/ui 作者） |
| **协议** | MIT |
| **增速** | 发布初期约 832 stars/天 |

**核心理念：用最强模型做"思考"，用便宜模型做"执行"。**

```
开发者 → /improve（昂贵模型审计+生成执行计划）
       → plans/ 目录（自包含的执行规格）
       → 便宜模型执行→测试→交付
```

**9 大审计类别**：正确性/安全/性能/测试覆盖率/技术债务/依赖迁移/开发者体验/文档/方向建议

**硬性规则**：绝不直接修改源代码（只写 `plans/`）、绝不运行会改变工作树的命令、绝不暴露密钥、被要求直接实现时会拒绝。

> 💡 **看点**：这代表了 AI 编程工具的进化方向——不是让 AI 取代工程师，而是用工作流设计最大化高智能模型的杠杆效应。
