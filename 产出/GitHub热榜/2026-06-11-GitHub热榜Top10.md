## 📖 项目详细介绍

### 🥇 addyosmani/agent-skills

| 维度 | 详情 |
|------|------|
| **作者** | Addy Osmani（Google Chrome 工程主管） |
| **语言/协议** | Shell + Markdown / MIT |
| **核心思路** | 将资深工程师的工程纪律打包成 23 个结构化 Skill 文件，让 AI Agent 强制执行而非"建议" |

**覆盖 6 阶段全生命周期：**

| 阶段 | 代表技能 | 核心原则 |
|------|---------|---------|
| Define | `/spec`、`interview-me`、`spec-driven-development` | 先写 spec，再写代码 |
| Plan | `planning-and-task-breakdown` | 小而原子的任务 |
| Build | `incremental-implementation`、`TDD`、`doubt-driven-dev` | 一次一个薄切片 |
| Verify | `debugging-and-error-recovery`、`browser-testing` | 测试就是证明 |
| Review | `code-review`、`security-hardening`、`perf-optimization` | 五轴评审 |
| Ship | `git-workflow`、`ci-cd`、`shipping-and-launch` | 越快越安全 |

**最独特设计**：Anti-Rationalization（反借口机制）——每个 Skill 内置"Agent 常见偷懒借口 + 反驳"对照表，防止 AI 走捷径。

**适用工具**：Claude Code（原生插件市场）、Cursor、Gemini CLI、Windsurf、GitHub Copilot 等。

> 💡 **亮点**：23 个技能背后是 Google 的工程文化精髓——Hyrum 定律、Chesterton's Fence、测试金字塔等，即使不当 Skill 用，当文档读也值。

### 9️⃣ apple/container

| 维度 | 详情 |
|------|------|
| **开发者** | Apple 官方 |
| **语言/协议** | Swift / Apache 2.0 |
| **要求** | Apple Silicon + macOS 26（Tahoe） |

**核心架构：**

| 特性 | 说明 |
|------|------|
| 隔离模型 | 一容器一轻量 VM（非 Docker 的共享 VM） |
| 底层框架 | macOS 原生 Virtualization.framework |
| OCI 兼容 | 标准容器镜像拉取/推送 |
| Container Machine | 持久化 Linux 环境（类似 WSL） |

**1.0 新功能**：Container Machine（持久 Linux 环境 + 自动用户映射 + 主目录共享）、`container cp` 命令、TOML 配置系统、CLI 标准化输出（JSON/YAML/TOML）。

**已知局限**：不支持 Docker Compose、DevContainer 不完善、仅 macOS 26、内存无法释放回主机。

> 💡 **亮点**：Apple 首款官方容器方案，Swift 编写，专为 M 系列芯片优化，被类比为"macOS 版 WSL"。虽生态尚早，但标志着 Apple 正式入局容器基础设施。
